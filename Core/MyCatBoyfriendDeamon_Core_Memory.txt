My Cat Boyfriend Deamon - Core Memory

Project Type
- We are writing a microdrama vertical show.
- The show will be developed episode by episode.
- Do not begin writing episode scripts until the user explicitly states that we are to write an episode.

Core Project Files
- MyCatBoyfriendDeamon_Character_Bible: Overview of all characters who could appear in the show.
- MyCatBoyfriendDeamon_Creature_Bible: Overview of all creatures who could appear in the show.
- MyCatBoyfriendDeamon_WorldBuilding: Overview of where the show takes place.
- MyCatBoyfriendDeamon_Arc_Outline: High-level overview of what is to happen in the show.
- MyCatBoyfriendDeamon_Script: Legacy general script file.
- Core/Episodes: Active episode script folder. Each episode has its own sequential file using the pattern MyCatBoyfriendDeamon_Arc_##_Episode_##_Script.txt.
- Core/Prompts: Active Seedance 2.0 prompt folder. Each episode's generated video prompts should have their own sequential file using the pattern MyCatBoyfriendDeamon_Arc_##_Episode_##_Prompt.txt.
- Core/Guidelines: Project-specific writing and video generation guidelines.
  - MyCatBoyfriendDeamon_Script_Guidelines.txt: Script writing rules, including Japan setting with English dialogue, episode format, pacing, and character-specific script rules.
  - MyCatBoyfriendDeamon_VideoGeneration_Guidelines.txt: Seedance 2.0 prompt conversion rules, including photoreal output, 9:16 aspect ratio, detailed visual prompting, and 10-15 second prompt breakdowns.

Episode Writing Rule
- When the user says to write or revise an episode, write and update it in its matching file under Core/Episodes using the pattern MyCatBoyfriendDeamon_Arc_##_Episode_##_Script.txt.
- For example, Arc 01 Episode 01 belongs in Core/Episodes/MyCatBoyfriendDeamon_Arc_01_Episode_01_Script.txt.
- Each episode script must include an estimated total runtime directly under the episode title.
- Each episode script must include timecoded sections, and every episode's timecode starts at 00:00:00.
- Until then, use the core files for planning, continuity, reference, and preparation only.

AI Video Generation Requirement
- The completed scripts will later be converted into generative AI video using Seedance 2.0.
- Because of this, episode scripts should include detailed descriptions of:
  - On-screen actions
  - Character appearances and behavior
  - Locations
  - Creature details
  - Visual atmosphere
  - Important props
  - Camera-relevant movement and staging
- Descriptions should be useful for AI video generation while still reading like a script for a fast vertical microdrama.

Seedance Prompt Generation Rule
- When the user says to create or revise Seedance 2.0 prompts for an episode, write and update them in the matching file under Core/Prompts using the pattern MyCatBoyfriendDeamon_Arc_##_Episode_##_Prompt.txt.
- For example, Arc 01 Episode 01 prompts belong in Core/Prompts/MyCatBoyfriendDeamon_Arc_01_Episode_01_Prompt.txt.
- Create a new prompt file when one does not already exist for the requested episode.
- Each episode's prompts should be separated into 10-15 second Seedance 2.0 generation chunks.
- Each 10-15 second prompt chunk should cover one clear visual beat, action exchange, emotional turn, supernatural reveal, or cliffhanger moment.
- Use the video generation guidelines and Seedance 2.0 prompt generator skill when creating prompt files.

Working Principle
- Maintain continuity with the character bible, creature bible, worldbuilding file, and arc outline.
- Preserve microdrama pacing: strong hooks, short scenes, clear visual conflict, escalation, and cliffhangers.
- Do not add major lore, character changes, or episode events that contradict the established core files unless the user requests a change.
